## Overview

Oracle is a browser-based 3D world explorer that replaces traditional input devices with your body. Using only a standard webcam, it tracks your head movement for look-around navigation and your hands for walking, world-switching, and a Spider-Man-style web-slinging teleportation mechanic -- all rendered inside photorealistic Gaussian splat environments generated by World Labs' Marble API.

The worlds themselves are AI-generated 3D scenes: a Manhattan Fifth Avenue streetscape, an ancient golden temple, a neon cyberpunk rooftop, a haunted house, a cozy forest cottage, and a spaceship interior. Each world is loaded as a compressed `.spz` Gaussian splat and rendered in real-time via the Spark renderer on top of Three.js.

An integrated Gemini 2.0 Flash Live connection turns Oracle into a conversational AI guide. Gemini can respond to voice or text, switch worlds based on conversational mood (it treats worlds as "dreams"), and fly the camera to points of interest -- all through function-calling over a persistent WebSocket.

## Features

- **Head Tracking Navigation** -- MediaPipe FaceMesh maps head yaw, pitch, and roll to camera rotation in real-time. VR-style quaternion application: position stays fixed, only view direction changes. Auto-calibrates on first face detection.

- **Dual Hand Tracking** -- MediaPipe Hands tracks both hands simultaneously. Right hand vertical position controls walk forward/backward. Left hand position maps to orbit rotation. Fist gesture (held 200ms) on either hand switches worlds.

- **Spider-Man Web-Slinging** -- Spider-Man hand pose (index + pinky extended, middle + ring curled) on the left hand shoots an animated web line with Bezier curve sag. Raycasts against collider meshes to stick to building surfaces. Releasing the pose triggers a smooth arc flight to the target with FOV widening for a speed-rush effect.

- **6 AI-Generated Worlds** -- Manhattan Fifth Ave (with collider mesh for wall collision), Ancient Temple, Cyberpunk Rooftop, Haunted House, Cozy Cottage, and Spaceship. Each with custom sky colors and camera positioning.

- **Wall & Floor Collision** -- GLB collider meshes loaded alongside splats. 4-directional raycasting pushes the camera back from walls. Floor clamping prevents falling through the ground. Web teleport targets are clamped above floor level.

- **Gemini Live AI Chat** -- Real-time WebSocket connection to Gemini 2.0 Flash. System prompt defines Oracle as a poetic, world-thinking AI. Function calling enables `change_world`, `fly_to`, and `get_live_data`. Voice input via Web Speech API with pulsing mic indicator.

- **3D Hand Skeleton Rendering** -- Both hands rendered as joint spheres and bone line segments directly in the Three.js scene, positioned in camera space with FOV-scaled mapping.

- **Webcam HUD Overlay** -- Picture-in-picture webcam feed with green scanline animation, corner brackets, face oval outline, calibration crosshair, and per-hand gesture labels drawn on a 2D canvas overlay.

- **World Generation CLI** -- Node.js scripts (`generate.js`, `gen-hq.js`) to create new worlds via the World Labs Marble API. Generates `.spz` splat files and optional collider meshes from text prompts.

- **Ambient Audio** -- Procedural drone audio using Web Audio API oscillators (60Hz, 90Hz, 45Hz triangle) with delay feedback for spatial depth.

- **Post-Processing Pipeline** -- Unreal Bloom pass via Three.js EffectComposer for atmospheric glow.

- **Oracle Entity** -- Morphable icosahedron with physical material (metalness 1.0, clearcoat, iridescence), procedural vertex displacement, and mood-reactive environment maps (dream, wrath, calm, void).

## Architecture

```
index.html
  |
  +-- src/main.js          Core render loop, scene setup, world switching,
  |                        gesture integration, camera flight, collision,
  |                        Spider-Man web mechanics, chat UI
  |
  +-- src/gestures.js      MediaPipe Hands + FaceMesh processing,
  |                        gesture state machines (fist, palm, spider-man),
  |                        head tracking calibration, webcam overlay drawing
  |
  +-- src/gemini.js        Gemini 2.0 Flash Live WebSocket connection,
  |                        function calling (change_world, fly_to, get_live_data),
  |                        system prompt defining Oracle personality
  |
  +-- src/entity.js        Morphable Oracle entity with physical material,
  |                        procedural env maps, mood system, vertex displacement
  |
  +-- src/particles.js     2500-particle orbital system with additive blending
  |
  +-- src/postfx.js        EffectComposer + UnrealBloomPass pipeline
  |
  +-- src/audio.js         Procedural ambient drone (oscillators + delay)
  |
  +-- worlds/*.json        Pre-generated world configs (splat URLs, collider URLs)
  |
  +-- generate.js          CLI: text prompt -> World Labs Marble API -> .spz world
  +-- gen-hq.js            CLI: high-quality Manhattan generation (Marble 0.1-plus)
```

**Data flow:**

1. Webcam frames are sent to MediaPipe Hands (every frame) and FaceMesh (every 2nd frame)
2. `gestures.js` processes landmarks into normalized control signals (yaw, pitch, roll, pull, fist, spider-man)
3. `main.js` smooths signals via EMA/lerp and applies them: head rotation as post-update quaternions, hand pull as forward/backward walk, fist as world switch, spider-man as web-sling teleport
4. Gaussian splats are loaded via Spark with an OpenCV-to-OpenGL quaternion flip (`quaternion.set(1,0,0,0)`)
5. Collider meshes (GLB) are loaded invisibly for raycasting (wall collision + web targeting)
6. Gemini Live receives text/voice input and can trigger world switches or camera flights via function calls

## Getting Started

### Prerequisites

- Node.js 18+
- A webcam (for gesture/head tracking)
- A Gemini API key (optional, for AI chat features)
- A World Labs API key (optional, for generating new worlds)

### Install

```bash
git clone https://github.com/gtrush03/oracle-world-explorer.git
cd oracle-world-explorer
npm install
```

### Run

```bash
npm run dev
```

Open the URL shown in terminal (default `http://localhost:5173`). Allow camera access when prompted.

### Controls

| Input | Action |
|-------|--------|
| Head turn left/right | Look left/right (yaw) |
| Head tilt up/down | Look up/down (pitch) |
| Head lean | Subtle camera roll |
| Right hand up/down | Walk forward/backward |
| Left hand position | Orbit rotation |
| Fist (either hand, hold 200ms) | Switch to next world |
| Spider-Man pose (left hand) | Shoot web -- release to fly to target |
| Mouse drag | Orbit rotation (OrbitControls) |
| Scroll wheel | Walk forward/backward |
| Chat input / mic button | Talk to Oracle (requires Gemini connection) |

### Generate New Worlds

```bash
WLT_API_KEY=your_key node generate.js "A cozy cyberpunk apartment with neon lights"
```

Worlds are saved to `worlds/` as JSON files containing splat URLs and metadata.

## How It Works

The key technical challenge is making head and hand tracking feel natural inside a 3D scene without fighting Three.js OrbitControls. Oracle solves this by applying head rotation as quaternions *after* `controls.update()` each frame -- yaw in world space (around Y axis), pitch in camera-local space (around X axis). This means OrbitControls handles mouse interaction cleanly while head tracking layers on top without position drift or axis flipping.

The Spider-Man web mechanic uses raycasting against invisible collider meshes to find building surfaces. When the gesture is detected, an animated Bezier curve web line extends from the fingertip to the hit point over 300ms with cubic ease-out. On release, the camera flies to the target in a 1.5-second arc trajectory with dynamic FOV widening at the peak for a speed sensation, then FOV snaps back on landing.

Gaussian splat worlds are loaded as compressed `.spz` files via the Spark renderer, which integrates with Three.js as a custom mesh type. The critical detail: Spark splats use OpenCV camera conventions, so a quaternion flip (`set(1,0,0,0)`) converts to OpenGL/Three.js conventions. Collider meshes use the same transform to stay aligned.
